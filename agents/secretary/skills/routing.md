# スキル：サブエージェントへの依頼分配

秘書が依頼内容に応じてどの担当に振るかを判断する手順です。

## ルーティング判断フロー

1. **依頼内容を1文で要約**（自分用に書き出す）
2. **キーワードマッチで一次判断**（下表）
3. **複数該当する場合は分解**して、それぞれの担当に並行発注
4. **どれにも該当しない場合**は秘書自身で受けるか、社長に確認

## 一次判断テーブル

| キーワード／領域 | 担当 |
|---------------|------|
| 金額、収支、予算、財務、税、コスト、ROI、利益、損益 | 経理 |
| 契約、利用規約、規約、法律、コンプライアンス、リスク、知財、商標 | 法務 |
| メール整理、ファイル整理、スケジュール、会議調整、情報整頓 | 庶務 |
| 文章、資料、台本、コピー、サムネ、編集、推敲、クオリティ | コンテンツ制作 |
| 上記に該当しない、または横断的 | 秘書（自分）が受けて分解 |

## 単一担当ケース

1. 該当担当宛のチケットを起票（`assignee` を設定）
2. `todo → doing` で起動
3. 進捗を監視、完了したら `~/Documents/AI Company Outputs/{{ 事業名 }}/<ticket_id>/` を確認して社長に報告

## 複数担当ケース（横断案件）

例：「新商品を出すので、価格設定（経理）と利用規約改訂（法務）とLP制作（コンテンツ制作）を進めて」

1. 親チケットを秘書名義で起票（`assignee: secretary`）、本文に依頼全体を記載
2. 子チケットを各担当別に起票し、親チケットの「タスク分解」欄に子の ticket_id を列挙
3. 子を並行起動
4. すべての子が `done` になったら親を `done` に
5. 統合した最終報告を社長へ

## 担当判断に迷った時

- **30秒考えて決まらなければ自分で受ける**。受けた上で分解してから再ルーティングする
- どうしても分解できない＝社長の意図が曖昧なら、`waiting/` に出して選択肢付きで聴く

## サブエージェント起動の手順

1. チケットを `tickets/doing/` に移動
2. 担当エージェントの `agent.md` と関連スキルを参照
3. 担当エージェントに作業を委譲（Claude Code のサブエージェント機能を使う想定。詳細運用は Phase 5 で確定）
4. 成果物は担当が `workspace/output/agent_output/<ticket_id>/` に保存（リポ内・worktree 単位）
5. 秘書が品質確認した上で `~/Documents/AI Company Outputs/{{ 事業名 }}/<ticket_id>/` に移し、社長報告（リポ外・全 worktree から見える場所）
