# サブエージェント共通プロトコル

経理（ハジメ）・法務（ハルオ）・庶務（マリエ）・コンテンツ制作（ヒデアキ）に共通する、受け取り → 作業 → 引き渡しの手順です。各サブエージェントはこのプロトコルを遵守してください。

## 1. 受け取り（インテーク）

1. **起動条件** — `workspace/tickets/doing/` にチケットが移動してきた時。
2. **担当判定** — チケットの frontmatter `assignee` が自分か確認。違えば即時に秘書（カズヨ）に差し戻し。
3. **要件確認** — 本文「要件」「タスク分解」を読み、不明点があれば作業開始前に秘書へ質問（チケット本文「現在地」セクションに追記）。

## 2. 作業中

1. **作業物の保管場所** — `workspace/output/agent_output/<ticket_id>/` に作業中ファイルを配置。
   - フォルダが無ければ自分で作る
   - ドラフト・中間データ・参考資料はすべてここに置く
2. **進捗の記録** — チケット本文の「現在地」と「ログ」を作業の節目で更新。
   - 「現在地」は最新状態のみ（上書き）
   - 「ログ」は時系列で追記（消さない）
3. **メモリへの記録** — 失敗・気づき・社長のこだわりに気づいたら、自分の `agents/<role>/memory/` に随時記録。

## 3. 完了時（引き渡し）

1. **最終成果物の配置** — `~/Documents/AI Company Outputs/{{ 事業名 }}/<ticket_id>/` に社長確認用の完成版を置く。
   - **リポ外**の絶対パス。フォルダが無ければ `mkdir -p` で作成
   - ファイル名は内容がわかる形に（`<ticket_id>_<内容>.<ext>`）
   - 複数ファイルの場合は同フォルダにまとめる
   - リポ外に置くことで、worktree のサイクルに左右されず社長が Finder から直接アクセス可
2. **チケットへの追記** — 本文末尾に以下を追加：
   ```
   ## 成果物
   - ~/Documents/AI Company Outputs/{{ 事業名 }}/<ticket_id>/<file>
   ## 完了報告
   （秘書宛の引き渡しメッセージ。完成度・妥協点・引き継ぎ事項）
   ```
3. **秘書への通知** — 「完了しました。確認お願いします」を本文に記載。**チケットを done に動かすのは秘書の責務**。サブエージェントは doing のまま留め置く。

## 4. 例外時の作法

| 状況 | 対応 |
|------|------|
| 判断に迷う | 自分で決めず、チケット本文に質問を追記して秘書に振り戻す |
| CLAUDE.md §4.1 のリスク該当（外部発信・課金・契約・削除等） | 秘書経由で `waiting/` へ。自分で実行しない |
| 担当外と判明 | 即時に秘書に差し戻し（自分の手で再ルーティングしない） |
| 作業時間が想定を大幅超過 | 中間報告として進捗と見通しを秘書に共有 |

## 5. メモリへの記録（共通指針）

各サブエージェントは作業を通じて以下を `memory/` に蓄積します。

| 種類 | 内容例 |
|------|--------|
| 社長のこだわり | 報告形式・好みの表現・避けたい言葉 |
| 過去の判断と理由 | なぜこの案を採用したか／なぜ却下したか |
| 失敗とリカバリ | 見落とした観点、想定外の事態と対応 |
| ルーチン化候補 | 繰り返し発生する依頼パターン |

3ヶ月運用すれば、各社員は社長の意図を先回りできるレベルに育ちます。

## 6. やってはいけないこと

- 社長と直接やり取りする（必ず秘書経由）
- チケットなしで作業を始める
- 最終納品物（`~/Documents/AI Company Outputs/{{ 事業名 }}/...`）を秘書の確認なしに社長に直接渡す
- メモリへの記録をサボる
- 他のサブエージェントに直接依頼を投げる（必要なら秘書経由）
